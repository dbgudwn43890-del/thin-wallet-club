# Change Log Rules

발행 후 콘텐츠 수정, 삭제, 정정 절차.
모든 변경은 기록이다. 실수도 기록한다.

---

## 변경 유형

| 유형 | 정의 | 대응 |
|------|------|------|
| A. 경미한 오류 | 오탈자, 포맷 오류, 링크 누락 | 캡션 수정 |
| B. 정보 오류 (부분) | 비용, 기간, 조건 일부 틀림 | 캡션 정정 + 스토리 공지 |
| C. 정보 오류 (핵심) | 마감일, 대상, 출처 완전히 틀림 | 삭제 + 정정 게시물 |
| D. 브랜드 위반 | 금지 표현, 광고 미표시 등 | 즉시 삭제 |
| E. 법적 문제 | 저작권, 명예훼손 등 | 즉시 삭제 + 기록 보존 |

---

## 변경 절차

### A — 경미한 오류

1. 인스타그램 캡션 직접 수정
2. `content/published_log.md` 에 수정 내용 기록
3. 수정일 기록

### B — 정보 오류 (부분)

1. 캡션 수정 (정확한 정보로 교체)
2. 스토리에 정정 공지 게시 (예: "이전 게시물 OO 정보 수정됐어요")
3. `content/published_log.md` 에 정정 내용 + 이유 기록
4. 향후 동일 유형 오류 방지를 위해 `ops/quality_rules.md` 체크리스트 점검

### C — 정보 오류 (핵심)

1. 게시물 즉시 삭제
2. 스토리에 정정 공지
3. 정확한 정보가 확인되면 새 게시물로 재발행
4. `content/published_log.md` 에 삭제 사유 + 재발행 여부 기록
5. 오류 원인 분석 → `ops/quality_rules.md` or `ops/human_review_rules.md` 보완

### D — 브랜드 위반

1. 즉시 삭제
2. `content/published_log.md` 에 위반 내용 + 삭제 사유 기록
3. `brand/forbidden_rules.md` 에 해당 케이스 추가
4. 에이전트 지시문 업데이트 검토

### E — 법적 문제

1. 즉시 삭제 (48시간 이내)
2. 삭제 기록 영구 보존
3. 재발행 금지

---

## 기록 형식

`content/published_log.md` 에 다음 형식으로 추가:

```
### [변경일] YYYY.MM.DD — {콘텐츠명} 수정

- 변경 유형: A / B / C / D / E
- 변경 내용:
- 변경 이유:
- 후속 조치:
- 재발 방지:
```

---

## 삭제 기록 보존 원칙

삭제된 게시물의 원고, 디자인 프롬프트, 성과 데이터는 삭제하지 않는다.
`data/archive/` 로 이동 후 파일명에 `_deleted` 접미사 추가.

```
예시: data/archive/001_culture_example_archive_deleted.yml
```

실수 기록은 시스템을 개선하는 자산이다.
