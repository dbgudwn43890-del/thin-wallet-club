# Quality Rules

단계별 품질 기준. 각 상태 전환 전에 아래 기준을 충족해야 한다.

---

## 후보 수집 품질 (idea → pre_selected)

| 기준 | 최소 요건 |
|------|---------|
| 출처 존재 여부 | Tier 1 or Tier 2 출처 확인 가능 |
| 타겟 적합도 | 서울/수도권 20대 대상임이 명확 |
| 비용 | 무료 or 3만 원 이하 (예외는 큐레이터 판단) |
| 시의성 | 발행 시점에 유효할 것 |

---

## 검증 품질 (verifying → verified)

| 기준 | 최소 요건 |
|------|---------|
| 공식 출처 | Tier 1 URL 접근 가능 + 내용 일치 확인 |
| 확인일 | 48시간 이내 (행사는 24시간 이내) |
| 필수 정보 6개 | 대상 / 지역 / 기간 / 비용 / 신청방법 / 주의사항 |
| TODO 항목 | 0개 (모두 해소) |

---

## 브리프 품질 (selected → briefed)

| 기준 | 최소 요건 |
|------|---------|
| 제목 후보 | 5개 이상 |
| 슬라이드 구성 | 8장 구조 완전 설계 |
| 핵심 약속 | 독자가 얻는 것이 1문장으로 명확 |
| 팩트 목록 | 확인된 것 / TODO 구분 완료 |

---

## 원고 품질 (briefed → scripted)

| 기준 | 최소 요건 |
|------|---------|
| 슬라이드 수 | 정확히 8장 |
| 제목 길이 | 각 장 15자 이내 |
| 본문 길이 | 각 장 4줄 이내 |
| TODO | 0개 |
| 출처/확인일 | 모든 정보 카드 하단에 표기 |
| 금지 표현 | 없음 (`brand/forbidden_rules.md` 기준) |
| 톤 | 얇은지갑클럽 에디터 톤 |

---

## 디자인 프롬프트 품질 (scripted → designed)

| 기준 | 최소 요건 |
|------|---------|
| 포맷 명시 | 1080x1350, 8장 |
| 무드 키워드 | editorial, magazine, minimal 포함 |
| 슬라이드별 텍스트 | 전체 포함 |
| 금지 요소 | 명시됨 (공공기관 색상, 과한 이모지 등) |
| 고정 표기 | 출처, 확인일, 브랜드명 위치 명시 |

## 검수 품질 (designed → in_review)

`ops/human_review_rules.md` A/B/C/D 항목 전체 체크리스트 통과.

## 최소 발행 주기

아래 기준은 정식 운영 단계에서만 적용한다.
현재 구조 세팅 단계에서는 후보 수집, 원고 작성, 디자인 프롬프트 생성을 시작하지 않는다.

| 상황 | 조건 |
|------|------|
| 정상 운영 | 주 4회 (월/수/금/일) |
| 후보 부족 | 에세이형으로 대체 (검증 불필요) |
| 검증 실패 | 해당 주 발행 취소 or 다른 후보로 대체 |
| 마감 D-3 이내 정보 | 리드타임 부족 → 다음 주로 연기 |

---

## 품질 실패 시 처리

| 실패 단계 | 처리 |
|---------|------|
| 검증 실패 | `data/rejected/` 로 이동 + 사유 기록 |
| 원고 반려 | Editor에게 반환 + 코멘트 포함 |
| 디자인 반려 | Design Director에게 반환 + 코멘트 포함 |
| 발행 후 오류 | `ops/change_log_rules.md` 절차 실행 |
