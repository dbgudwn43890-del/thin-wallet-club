## 후보 수집 품질 (idea → pre_selected)

| 기준 | 최소 요건 |
|------|---------|
| 출처 존재 여부 | Tier 1 or Tier 2 출처 확인 가능 |

## 전시/행사 일정 판단 기준

전시/행사 후보는 종료일이 가깝다는 이유만으로 자동 폐기하지 않는다.
가까운 종료일은 리스크이면서 동시에 저장/공유를 유도하는 긴급성 요소가 될 수 있다.
Curator는 `timing_fit`과 `urgency_value`를 함께 판단한다.

| 일정 상태 | 기준 | 처리 |
|-----------|------|------|
| `expired` | 기준일에 이미 종료됨 | 제외 |
| `last_chance` | 종료까지 0~3일 남음 | 카드뉴스 메인 후보에서는 기본 제외. 스토리/긴급 알림/오늘의 추천 후보로만 사용 가능 |
| `ending_soon` | 종료까지 4~7일 남음 | 카드뉴스 후보로 사용 가능. "이번 주 안에 가야 함", "곧 종료" 표기 필수 |
| `ongoing_near` | 종료까지 8~21일 남음 | 카드뉴스 후보로 적합. 발행 리드타임을 고려해 안정적으로 사용 가능 |
| `ongoing_stable` | 종료까지 22일 이상 남음 | 안정적 후보. 단, 긴급성은 낮을 수 있음 |
| `upcoming_soon` | 아직 시작 전이고 30일 이내 시작 | 카드뉴스 후보로 사용 가능. "곧 열림", "미리 저장", "오픈 예정" 표기 |
| `upcoming_later` | 시작일까지 31일 이상 남음 | 보류 후보. 공식 정보가 있어도 즉시 제작보다 재검토 권장 |

### 시작 전 후보

아직 시작 전인 후보도 공식 출처에서 기간, 장소, 비용, 무료 여부가 확인되면 사용할 수 있다.
단, 시작일이 30일보다 많이 남은 경우에는 `upcoming_later`로 두고 보류한다.

### Curator timing 필드

Curator 출력에는 가능하면 아래 필드를 포함한다.

```yaml
timing_fit: 0~5
urgency_value: 0~5
timing_label: ongoing_stable | ending_soon | last_chance | upcoming_soon | upcoming_later | expired
timing_note: "일정 판단 근거"
```

`timing_fit`은 발행 가능성과 일정 안정성을 본다.
`urgency_value`는 지금 저장하거나 공유해야 할 이유를 본다.
종료일이 가깝다는 이유만으로 `timing_fit`을 0점 처리하지 않는다.
다만 `last_chance`는 카드뉴스 메인 후보가 아니라 스토리/긴급 알림 용도로 분리한다.

## 검증 품질 (verifying → verified)

| 기준 | 최소 요건 |
|------|---------|
| 공식 출처 | Tier 1 URL 접근 가능 + 내용 일치 확인 |
| 확인일 | 48시간 이내 (행사는 24시간 이내) |
| 필수 정보 6개 | 대상 / 지역 / 기간 / 비용 / 신청방법 / 주의사항 |
| TODO 항목 | 0개 (모두 해소) |

## 원고 품질 (briefed → scripted)

| 기준 | 최소 요건 |
|------|---------|
| 슬라이드 수 | 정확히 8장 |
| 제목 길이 | 각 장 15자 이내 |
| 본문 길이 | 각 장 4줄 이내 |
| TODO | 0개 |
| 출처/확인일 | 모든 정보 카드 하단에 표기 |
| 금지 표현 | 없음 (`brand/forbidden_rules.md` 기준) |
| 톤 | 얇은지갑클럽 에디터 톤 |
| 커버 문구 | 명확한 정보 중심. 쿨해 보이려는 말투/내부 기획어 금지 |

### 원고 톤 품질 기준

카드뉴스 원고는 담백하고 정확한 정보가 먼저 보여야 한다.
감도는 표현의 과장이 아니라 편집, 여백, 정보 위계, 디자인으로 만든다.

금지:
- 쿨해 보이려는 말투
- 브랜드가 자기 감도를 과하게 의식하는 말투
- 내부 기획어를 독자-facing 문구처럼 쓰는 방식
- 정보보다 분위기를 앞세우는 말투
- 과장된 추천어

커버는 메인 정보 + 짧은 보조 정보로 구성한다.
예: "서울 무료 전시 5", "지금 볼 수 있는 무료 전시", "저장해둘 서울 전시"

## 디자인 프롬프트 품질 (scripted → designed)

| 기준 | 최소 요건 |
|------|---------|
| 포맷 명시 | 1080x1350, 8장 |
| 무드 키워드 | editorial, magazine, minimal 포함 |
| 슬라이드별 텍스트 | 전체 포함 |
| 금지 요소 | 명시됨 (공공기관 색상, 과한 이모지 등) |
| 고정 표기 | 출처, 확인일, 브랜드명 위치 명시 |
| 카피 톤 유지 | 원고의 정보 중심 톤을 유지하고 금지 문구를 새로 만들지 않음 |

## 최소 발행 주기

아래 기준은 정식 운영 단계에서만 적용한다.
현재 구조 세팅 단계에서는 후보 수집, 원고 작성, 디자인 프롬프트 생성을 시작하지 않는다.

| 상황 | 조건 |
|------|------|
| 정상 운영 | 주 4회 (월/수/금/일) |
| 후보 부족 | 에세이형으로 대체 (검증 불필요) |
| 검증 실패 | 해당 주 발행 취소 or 다른 후보로 대체 |
| 종료 D-0~D-3 정보 | 카드뉴스 메인 후보 제외. 스토리/긴급 알림/오늘의 추천 후보로만 사용 |
| 종료 D-4~D-7 정보 | 카드뉴스 후보 가능. "곧 종료" 표기 필수 |

| 실패 단계 | 처리 |
|---------|------|
| 검증 실패 | `data/rejected/` 로 이동 + 사유 기록 |
| 원고 반려 | Editor에게 반환 + 코멘트 포함 |
| 디자인 반려 | Design Director에게 반환 + 코멘트 포함 |
