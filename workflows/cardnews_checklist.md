# Card News Upload Checklist

업로드 직전 최종 검수. 전 항목 통과해야 업로드.

---

## 정보 검수

- [ ] 모든 슬라이드에 확인일이 있는가?
- [ ] 모든 슬라이드에 출처가 있는가? (또는 마지막 장에 일괄 표기)
- [ ] 마감일이 지나지 않았는가?
- [ ] 서울/수도권 대상이 맞는가?
- [ ] 대상 조건이 빠지지 않았는가?
- [ ] 비용 정보가 정확한가?
- [ ] 신청 방법이 현재 유효한가?

---

## 브랜드 검수

- [ ] 얇은지갑클럽 톤인가?
- [ ] 싸구려 느낌이 아니라 저예산 취향 느낌인가?
- [ ] 공공기관 카드뉴스처럼 보이지 않는가?
- [ ] 과장 표현이 없는가?
- [ ] 금지 표현이 없는가? (`brand/forbidden_rules.md` 참조)

---

## 디자인 검수

- [ ] 1080x1350 비율이 맞는가?
- [ ] 8장 흐름이 자연스러운가? (커버 → 정보 → CTA)
- [ ] 첫 장이 저장하고 싶게 생겼는가?
- [ ] 마지막 장에 저장/팔로우 CTA가 있는가?
- [ ] 한 장에 정보가 너무 많지 않은가? (5줄 이내)
- [ ] 제목/본문 위계가 명확한가?
- [ ] 출처/확인일이 각 장에 보이는가?
- [ ] 브랜드명 "얇은지갑클럽"이 마지막 장에 있는가?

---

## 캡션 검수

- [ ] 첫 줄에 핵심 정보 or 훅이 있는가?
- [ ] 조건 요약이 포함되어 있는가?
- [ ] 저장/팔로우 CTA가 있는가?
- [ ] 해시태그가 15개 이내인가?
- [ ] 출처와 확인일이 캡션에 포함되어 있는가?

---

## 업로드 설정

- [ ] 릴스가 아닌 캐러셀 포맷 선택
- [ ] 위치 태그: 서울 (or 해당 지역)
- [ ] 공유 계정 (협업 계정) 설정 확인
- [ ] 예약 업로드 시간 확인 (권장: 오전 7시 or 오후 7시)

---

## 업로드 후 즉시

- [ ] `content/published_log.md`에 발행 기록
- [ ] 파일 → `outputs/published/`로 이동
- [ ] 첫 30분 댓글 반응 확인

---

## 오류 발생 시

정보 오류 발견 → 캡션 수정 공지 → 필요 시 삭제 후 재업로드
디자인 오류 → 재업로드 (24시간 이내)
심각한 사실 오류 → 즉시 삭제 + 정정 공지
